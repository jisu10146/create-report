> AI팀 개발 핸드오프용. **어떤 광고 성과 데이터(매체/캠페인 × 일자 × 비용/노출/클릭/전환)가 들어와도 동일한 8개 슬롯**으로 구성되는 범용 리포트.
>
> 관련 파일:

## 1. 입력 데이터

### 1.1 입력 형식
CSV 또는 JSON. 1행 = 1(일자 × 매체 × 캠페인)의 집계 데이터.

### 1.2 컬럼 정의

#### 필수 (없으면 리포트 생성 불가)

| 컬럼 | 타입 | 설명 | 예시 |
|------|------|------|------|
| `날짜` (date) | date | 집계 일자 | 2025-03-01 |
| `매체` (channel/media) | string | 광고 매체 | 구글, 네이버, 메타, 카카오 |
| `캠페인명` (campaign) | string | 캠페인 식별자 | [구글]_브랜드검색_CPC |
| `노출수` (impressions) | integer | 일일 노출 수 | 27099 |
| `클릭수` (clicks) | integer | 일일 클릭 수 | 1652 |
| `비용` (cost) | integer | 일일 광고비 (원) | 1568093 |

#### 선택 — 있으면 해당 분석 활성화

| 컬럼 | 타입 | 설명 | 없을 때 영향 |
|------|------|------|-------------|
| `전환수` (conversions) | integer \| null | 일일 전환 수 | CPA·CVR 분석 불가, 섹션 2·4·5·7에서 fallback 메시지 |
| `매출` (revenue) | integer | 일일 매출 | ROAS 분석 비활성 (없으면 CPA 중심으로 진행) |
| `디바이스` (device) | string | 모바일/PC | 디바이스별 교차 분석 비활성 |
| `타겟_세그먼트` (audience) | string | 타겟 세그먼트 | 타겟 세그먼트별 비교 비활성 |

### 1.3 컬럼 → 섹션 강화 매핑

| 리포트 섹션 | 필수 컬럼만으로 가능한 것 | 선택 컬럼이 있으면 강화 |
|------------|----------|---------------------|
| Executive Summary | 노출·클릭·비용 총합, CTR, CPC | 전환수 → CPA·CVR topMetrics 활성 |
| 핵심 KPI (MetricHighlight) | CTR·CPC + 매체 편차 | 전환수 → CPA·CVR 지표 추가 |
| 매체 비용 구성 (DonutChart) | 매체별 비용 점유율 | 항상 채워짐 |
| 매체 단면 (DataTable) | 비용, 노출, 클릭, CTR, CPC | 전환수 → CVR·CPA 행 추가 |
| 캠페인 순위 (DataTable) | 비용·CTR·CPC 기준 순위 | 전환수 → CPA 기준 TOP/BOTTOM 재정렬 |
| 주별 추이 (TrendLineChart) | 주별 매체 CTR·CPC 추이 | 전환수 → CPA 추이 추가 |
| 근본 원인 (InsightCard) | 매체·캠페인·피로도 3차원 진단 | 전환수 → 측정 공백 차원 활성 |
| 예산 재배분 (StrategyTable) | 매체·캠페인 기반 액션 | 전환수 → 낭비 금액 정량화 |

### 1.4 최소 권장 건수
- **100행 이상**: 매체별 일 단위 추이 분석이 유의미 (매체 4개 × 25일+)
- **300행 이상**: 캠페인별 주별 교차가 안정적
- **30행 미만**: 리포트 생성 가능하나 "통계적 신뢰도 낮음" 경고 표시

### 1.5 데이터 기간 요구
- **4주 이상 권장** — 주별 추이(섹션 6)가 의미를 가지려면 최소 3주
- **4주 미만**: TrendLineChart를 MetricHighlight(일별 변동계수)로 대체

## 2. 설계 원칙

### 단일 골격
광고 성과 데이터는 매체 구성·캠페인 이름·전환 정의가 도메인마다 다양하지만, **리포트 골격은 동일**하다. 슬롯에 들어가는 *매체 이름*과 *캠페인 이름*만 달라진다.

| 변하는 것 | 변하지 않는 것 |
|---|---|
| 매체 종류 (구글/네이버/메타 vs TikTok/Twitter) | "매체별 비용 구성" 섹션 |
| 캠페인 네이밍 규칙 | 캠페인 TOP/BOTTOM 비교 슬롯 |
| 전환 정의 (구매/가입/앱설치) | "CPA 기준 효율" 해석 프레임 |
| 기간 길이 (4주/3개월/1년) | 시계열 추이 섹션의 존재 |

### "필수 슬롯" 원칙
- 8개 섹션은 **항상 자리에 있다**. 데이터 부족 시 채우는 콘텐츠가 줄어들 뿐, 슬롯 자체는 유지.
- 전환수 부재 시 → "전환 데이터 없음, CTR·CPC 중심으로 분석" 명시 (슬롯 유지).
- AI팀이 구현 시 **섹션을 동적으로 빼지 않는다** — 일관된 사용자 경험.

---

## 3. 리포트 골격 — 8개 섹션 (고정)

```
1. Executive Summary           — 전체 요약
2. 핵심 효율 KPI               — CPA/CTR/CVR + 매체 편차
3. 매체 비용 구성              — 매체별 비용 점유율
4. 매체 효율 단면              — 매체 × 지표 교차 테이블
5. 캠페인 순위                 — 캠페인 10개 CPA·CTR·CVR 비교
6. 주별 추이                   — 매체별 주별 CPA·CTR 변화
7. 근본 원인                   — 매체 적합도·크리에이티브·측정 3차원
8. 예산 재배분                 — 즉시/단기/중기
```

이 순서는 **고정**. PM/Strategy Writer는 이 순서를 임의로 바꾸지 않는다.

## 4. 섹션별 슬롯 명세

### 섹션 1. Executive Summary
- **컴포넌트**: `ExecutiveSummary`
- **목적**: 리포트를 1분 안에 파악할 수 있게 핵심만 압축

| 슬롯 | 역할 | 데이터 부재 시 |
|---|---|---|
| `description` | 분석 기간·매체 수·캠페인 수·총 비용 규모의 메타 설명 | 항상 채워짐 |
| `topMetrics[0]` | **총 광고비** (데이터 볼륨) | 항상 |
| `topMetrics[1]` | **총 전환수** | 전환 부재 시 "총 클릭수"로 대체 |
| `topMetrics[2]` | **평균 CPA** | 전환 부재 시 "평균 CPC"로 대체 |
| `topMetrics[3]` | **매체 간 CPA 편차 배수** | 전환 부재 시 "매체 간 CPC 편차 배수" |
| `keyFindings[0]` | 현황 진단 (전체 효율 + 벤치마크 대비) | 항상 |
| `keyFindings[1]` | 가장 비효율적인 매체 또는 캠페인 | 항상 |
| `keyFindings[2]` | 시계열 악화 징후 (크리에이티브 피로 등) | 4주 미만이면 "변동성 큰 지표" |
| `keyFindings[3]` | 해결 방향 (재배분 요약) | 항상 |

- **콘텐츠 규칙**:
  - `description`은 **두괄식 미적용** — 리포트 메타 설명
  - `keyFindings`는 두괄식 + 4개가 순서대로 읽히면 하나의 이야기
  - 수치 개수 4개 슬롯 제한 ([feedback_executive_number_count.md] 준수)

---

### 섹션 2. 핵심 효율 KPI
- **컴포넌트**: `MetricHighlight` (items 3개)
- **목적**: 전체 광고 효율을 가장 강조할 3개 지표로 표현

| 슬롯 | 역할 | 채우는 예시 |
|---|---|---|
| `items[0]` | CPA + 업계 벤치마크 | "평균 CPA ₩15,022 (업계 ₩10K~30K)" |
| `items[1]` | CTR + 매체 편차 | "전체 CTR 3.67% (매체별 0.91~6.65%)" |
| `items[2]` | CVR + 상위·하위 비교 | "전체 CVR 5.47% (카카오 1.20% vs 구글 6.85%)" |

- **각 item 필드 규칙**:
  - `label`: KPI 이름
  - `value`: 실제 수치 (두괄식 미적용)
  - `sub`: 벤치마크 또는 비교 기준 1줄
  - `description`: 두괄식 인사이트 — 수치가 왜 중요한지, 매체 편차, 격차의 의미

- **데이터 부재 시 fallback**:
  - 전환수 부재 → CPA/CVR 카드 대신 CPC + 노출당 비용(CPM) 카드로 대체

---

### 섹션 3. 매체 비용 구성
- **컴포넌트**: `DonutChart`
- **목적**: 매체별 비용이 어떻게 분배되고 있는지 구성비로 제시

| 슬롯 | 역할 |
|---|---|
| `title` | "매체별 비용 구성" |
| `items` | 매체 목록 (비용 내림차순) + 각 매체 비용 금액 |

- **각 item 필드**:
  - `label`: 매체명
  - `value`: 비용 (원 단위, 숫자)
  - `count`: 비용 금액 텍스트 (예: "₩113,352,959" — optional)

- **섹션 라벨 규칙**: 1위 매체 집중도 + 의미
  - 예: "구글이 전체 광고비의 59.9%를 차지하며 단일 매체 집중도가 높은 구조"

- **레이아웃 규칙**: DonutChart는 바로 다음 DataTable과 2열 그리드로 묶임 (SingleSectionLayout에서 자동)

---

### 섹션 4. 매체 효율 단면
- **컴포넌트**: `DataTable`
- **목적**: 매체별 핵심 효율 지표를 한 표로 비교

| 슬롯 | 역할 |
|---|---|
| `columns` | 매체 목록 (비용 내림차순) |
| `rows` | 핵심 지표 (있는 컬럼만 행으로 추가) |
| `showDots` | `false` |

- **배치 근거**: 매체는 도메인상 보통 3~6개 고정이므로 columns에 가로 배치해도 overflow 위험 없음
- **행 구성 우선순위**:
  1. **비용** (항상 첫 행)
  2. **노출수**
  3. **CTR**
  4. **CPC**
  5. **전환수** — 전환 컬럼 있을 때
  6. **CVR** — 전환 컬럼 있을 때
  7. **CPA** — 전환 컬럼 있을 때

- **섹션 라벨 규칙**: 최악 매체의 CPA·CTR vs 최고 매체 대비 배수
  - 예: "카카오 CPA ₩33,019로 구글(₩13,598) 대비 2.4배 — 비용 점유 2.5%지만 효율 낭비 구간"

---

### 섹션 5. 캠페인 순위
- **컴포넌트**: `DataTable`
- **목적**: 개별 캠페인 단위에서 확대·중단 결정 가능한 정보 제공

| 슬롯 | 역할 |
|---|---|
| `columns` | **지표 목록 (고정 5개)** — 비용 / CTR / CVR / 전환수 / CPA |
| `rows` | **캠페인 목록 (가변 N개)** — `metric` 필드에 캠페인명, CPA 오름차순 정렬 |

- **구조 원칙 — Transpose 필수**:
  - 캠페인 수는 데이터에 따라 가변(5개·10개·50개 등)이므로 **rows**에 배치 (세로 증가로 수용)
  - 지표 수는 고정이므로 **columns**에 배치
  - 반대로 구성(캠페인을 columns에)하면 20개 이상일 때 가로 오버플로로 레이아웃이 깨짐 — data-agnostic structure 위반

- **표시 규칙** (이 에이전트 고유 결정 — CPA 줄 세우기 + 확대/중단 양극단 의사결정 목적):
  - **TOP 5 + BOTTOM 5 분리 (한 테이블 안에 연속 배치, 중간 생략)**
    - TOP 5: CPA 오름차순 상위 5 (효율 우수, 확대 대상)
    - BOTTOM 5: CPA 내림차순 상위 5 (효율 낮음, 중단·재설계 대상)
    - 중간 캠페인은 생략 (요약 행 없음) — 광고 도메인에서 중간 구간은 개별 액션 불필요
  - 캠페인 10개 이하면 TOP 5 + BOTTOM 5 = 전체와 자연스럽게 일치
  - 캠페인 11개 이상이면 중간 생략, headline에 "전체 N개 중 효율 상위 5 + 하위 5" 맥락 포함
  - 전환수 0건 캠페인 표기: "0건" (숨기지 않음)
  - 전환수 컬럼 부재 시: CPA/CVR/전환수 column 제거, 비용/노출/클릭/CTR/CPC 5개로 대체

- **섹션 라벨 규칙**: TOP·BOTTOM 매체 + 캠페인 간 CPA 편차 배수
  - 예: "구글 3개 캠페인이 TOP3를 차지하고 카카오 2개 캠페인이 BOTTOM2에 자리하며, 캠페인 간 CPA 격차가 최대 2.7배까지 벌어져 있습니다"

---

### 섹션 6. 주별 추이
- **컴포넌트**: `TrendLineChart`- **목적**: 4주 이상 기간에서 매체별 CPA·CTR의 주별 변화 추이 제시

| 슬롯 | 역할 |
|---|---|
| `title` | "매체별 주별 CPA 추이" |
| `xLabels` | 주차 라벨 (예: W1~W4) |
| `series` | 매체 4개 각각의 CPA 값 배열 |
| `benchmarks` | 전체 평균 CPA 기준선 |

- **데이터 부재 시 fallback**:
  - 4주 미만 → MetricHighlight로 대체하거나 일별 변동계수 요약으로 축소
  - 전환수 부재 → series를 CPC 또는 CTR로 변경

- **섹션 라벨 규칙**: 가장 두드러진 변화 매체 + 변화 방향
  - 예: "W4에서 카카오 CPA ₩45,534로 W3 대비 2.3배 악화, 메타 CTR은 W2 2.18%→W4 1.65%로 피로도 누적"

### 섹션 7. 근본 원인
- **컴포넌트**: `InsightCard` (items **정확히 3개**)
- **목적**: 위 섹션에서 본 비효율의 구조적 원인을 3개 차원으로 진단

| 슬롯 | 차원 | 전환수 있을 때 예시 | 전환수 없을 때 fallback |
|---|---|---|---|
| 카드 1 | **매체 적합도** — 매체와 목표 불일치 | "카카오는 구매 유도형 광고에 부적합 — CVR 1.1%" | CVR 대신 CTR 기준으로 판단 — "카카오 CTR 0.91%로 도달 효율 최저" |
| 카드 2 | **크리에이티브 피로** — CTR·CVR 시계열 하락 | "메타 CTR이 2주 차 2.18%→4주 차 1.65%로 피로 누적" | 동일 (CTR만으로 충분) |
| 카드 3 | **측정 환경** (전환수 있을 때) / **매체 집중 리스크** (전환수 없을 때) | "전환 결측 32%의 카카오는 최적화 신호 부족" | "구글 1개 매체에 비용 59.9% 집중 — 매체 정책·경쟁 변화에 구조적 취약" |

- **각 카드 규칙**:
  - `badge`: 차원명 ("매체 적합도" / "크리에이티브" / "측정 환경")
  - `title`: **명사구 헤드라인** (1줄, ~20자, 동사 종결 금지)
  - `description`: 근거 — 수치, 매체, 비교
  - `interpretation`: **인과 체인** (원인 → 결과 → 영향). title/description 재진술 금지

- **콘텐츠 규칙**:
  - 카드 간 내용 겹침 금지 — 3개 카드가 서로 다른 차원
  - 캠페인명 직접 인용은 description에서만, title에 금지

---

### 섹션 8. 예산 재배분
- **컴포넌트**: `StrategyTable`
- **목적**: 근본 원인 3차원에 대응하는 즉시/단기/중기 액션

| 슬롯 | 역할 | 행 수 |
|---|---|---|
| `immediate` | 캠페인 pause·예산 재배분 등 즉시 실행 | 1-2개 |
| `short` | 크리에이티브 교체·입찰 구조 개선 | 1-2개 |
| `mid` | 측정 환경 구축·매체 믹스 재설계 | 1-2개 |

- **각 행 필드**:
  - `strategy`: 행동 지향 1줄 (예: "카카오 2개 캠페인 중단 및 255만 원 재배분")
  - `objective`: 어떤 지표를 어떻게 개선하는지 (수치 근거 포함)
  - `actionPlan`: 실행 방법 1-2줄
  - `expectedImpact`: 정량 목표 (예: "전체 CPA ₩15,022→₩12,500")

- **콘텐츠 규칙**:
  - 모든 strategy는 섹션 7(근본 원인) 카드와 1:1로 연결 (원인 → 해결책)
  - 마지막 mid 항목은 **종합 KPI 목표** (CPA 또는 ROAS 개선 등)

## 5. 슬롯 채우기 로직

### 속도 대표값 슬롯 (Section 1 `topMetrics[2]`)
```
1순위: 전환수 컬럼 → "평균 CPA ₩X"
2순위: 전환수 부재 → "평균 CPC ₩X"
```

### 편차 지표 슬롯 (Section 1 `topMetrics[3]`)
```
1순위: 전환수 있음 → "매체 간 CPA 편차 X배"
2순위: 전환수 부재 → "매체 간 CPC 편차 X배"
```

### 매체 효율 단면 행 슬롯 (Section 4 `rows`)
```
순서: 비용 → 노출수 → CTR → CPC → (전환수 있을 때) 전환수 → CVR → CPA
```

### 주별 추이 series 슬롯 (Section 6)
```
1순위: 전환수 있음 → CPA 추이
2순위: 전환수 부재 → CPC 추이 or CTR 추이
```

### 근본 원인 3차원 슬롯 (Section 7)
```
카드 1 (매체 적합도):
  1순위: 전환수 있음 → CVR·CPA 기준으로 매체 적합도 판단
  2순위: 전환수 부재 → CTR 기준으로 매체 적합도 판단

카드 2 (크리에이티브 피로): CTR 시계열로 판단 — 전환수 유무 무관

카드 3 (측정 환경 vs 매체 집중 리스크):
  1순위: 전환수 있고 매체별 결측률 20%+ → "측정 환경" (전환 태깅 공백)
  2순위: 전환수 부재 or 결측률 낮음 → "매체 집중 리스크" (단일 매체 의존도·매체 믹스 편중)
```

## 6. 콘텐츠 규칙 (Ad Performance 특수)

> 글로벌 콘텐츠 규칙(두괄식, 명사구 헤드라인, ES.description=메타 설명, InsightCard 세 필드 역할 분리) → [sample-generator.md](../../sample-generator.md) §0~§0-4 참조

### 6.1 매체명·캠페인명 인용
- 매체명(구글/네이버/메타/카카오)은 label·description 어디서든 그대로 사용 가능
- 캠페인명(예: "[구글]_브랜드검색_CPC")은 **description 또는 DataTable에서만** 사용 (label/headline에 통째로 넣지 않음 — 가독성 저하)

### 6.2 수치 단위
- 비용: 원 단위 `₩1,234,567` (업계 관행 유지)
- 비율: % 단위 (CTR/CVR)
- CPA/CPC: `₩1,234` (단위 단수, 반올림 정수)

### 6.3 전환 결측 표현
- 캠페인 단위 결측률 20% 이상이면 InsightCard 측정 환경 카드에 정량화 (예: "전환 결측 32%")
- 섹션 5 캠페인 순위에는 결측을 제외한 실제 전환값 기준 표시, 주석으로 결측일수 표기 가능

### 6.4 컴포넌트 배치
- DonutChart(섹션 3) + DataTable(섹션 4)는 2열 그리드로 자동 묶임 (CLAUDE.md 규칙)
- DataTable이 연속 2개 오는 건 OK (섹션 4 매체 단면 → 섹션 5 캠페인 순위)
- 섹션 5·6은 정보량이 많아 전폭 레이아웃에 배치 (SingleSectionLayout의 자동 처리)
